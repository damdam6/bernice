1요약

설계 문서(docs/)와 구글 시트 데이터 레이어(시딩·보호, scripts/)만 있는 레포에 실제 웹앱 뼈대를 세우는 P0 단계. 레포 루트에 Vite + React + TypeScript 앱을 스캐폴딩하고, Tailwind v4에 농구 오렌지 디자인 토큰을 심고, functions/api/health.ts 헬스체크와 wrangler.jsonc로 Cloudflare Pages 배포 준비를 마친다. 로컬에서는 vite(5173) + wrangler pages dev(8788, /api 프록시) 이중 서버로 빈 홈과 /api/health가 함께 구동되는 것까지 확인한다.

2현황 분석

  • 레포 구성 — 추적 파일은 docs/(설계 HTML 4개), scripts/seed-sheet.mjs · scripts/protect-sheet.mjs, .gitignore뿐. 프론트/백엔드 앱 코드·package.json 없음.
  • .gitignore 이미 준비됨 — .gitignore:1-17에 secrets/, .dev.vars, .env*, node_modules/, dist/, .wrangler/가 전부 등재. 완료 조건의 "gitignore 반영"은 신규 작성이 아니라 git check-ignore 검증만 필요.
  • 서비스계정 키 위치 — 메인 체크아웃 secrets/bernice-502106-02d0840dab2f.json에 존재(untracked라 이 워크트리엔 없음). .dev.vars 생성 시 여기서 복사해야 함. 문서(docs/sheet-integration.html)의 secrets/sa-key.json과 실제 파일명이 다른 점 주의.
  • SHEET_ID는 비밀 아님 — scripts/seed-sheet.mjs:18에 이미 하드코딩·커밋됨(시트 자체는 비공개 + 서비스계정 공유). wrangler.jsonc의 평문 vars로 둬도 무방.
  • 확정된 아키텍처 — docs/sheet-integration.html §04: Pages Functions가 서비스계정 JWT로 시트를 읽고, /api/login(패스코드→HttpOnly 쿠키) · /api/records · /api/refresh를 제공. P0는 이 구조가 올라갈 자리(functions/)만 만들면 됨.
  • 디자인 토큰 근거 — 농구 오렌지는 이미 두 곳에 존재: 시트 헤더색 scripts/seed-sheet.mjs:64 ≈ #E8623A, 문서 액센트 docs/build-plan.html:10 #ff7a45. Tailwind 토큰은 이 계열로 스케일 구성.
  • 툴체인 — Node v24.11.0 · npm 11.6.1 (Vite 7 요건 충족). wrangler는 미설치 → devDependency로 추가.
  • 패스코드/관리자코드 값 미정 — docs/build-plan.html:143 블로커. P0에선 .dev.vars에 자리(placeholder)만 만들면 되고 실값은 P2 전에 결정.

3설계

  • 레포 루트 = 앱 루트 (모노레포 아님) — Vite 엔트리 index.html·package.json·vite.config.ts를 루트에 두고 src/·functions/를 추가, 기존 docs/·scripts/는 그대로 공존. Cloudflare Pages가 루트 빌드(dist/)를 바로 집는 가장 단순한 구조.
  • Tailwind v4 (CSS-first) — @tailwindcss/vite 플러그인 + src/index.css의 @import "tailwindcss"·@theme으로 토큰 정의. tailwind.config.js·PostCSS 설정 불필요. 토큰: 오렌지 프라이머리 스케일(#ff7a45/#e8623a 기반), 옅은 틴트(#FBE7DD), 보조 블루(#4a90ff), 시맨틱 good/warn/bad — 설계 문서 팔레트와 동일 계열. 기각안: Tailwind v3 + config 파일 — 신규 프로젝트에서 굳이 구버전 문법을 쓸 이유 없음.
  • Pages Functions 파일 라우팅 — functions/api/health.ts 하나로 뼈대 확립(GET /api/health → { ok: true, service: "bernice", ts }). P1의 records, P2의 login/refresh가 같은 폴더에 얹힘. 타입은 @cloudflare/workers-types devDep + functions/tsconfig.json.
  • wrangler.jsonc — name: "bernice", pages_build_output_dir: "./dist", compatibility_date, 평문 vars.SHEET_ID. 시크릿 3종(서비스계정 키 JSON·팀 패스코드·관리자 코드)은 로컬 .dev.vars / 프로덕션 Pages Secret으로만.
  • 로컬 개발 = 이중 서버 + 프록시 — ① vite(5173, HMR) ② wrangler pages dev dist --port 8788(Functions + 빌드 산출물). vite.config.ts의 server.proxy로 /api → localhost:8788 위임. npm scripts로 dev(vite) · dev:api(wrangler) · dev:all(둘 동시 실행) 제공. 기각안 1: wrangler pages dev -- vite 프록시 모드 — deprecated. 기각안 2: @cloudflare/vite-plugin — Workers 전용이라 Pages Functions 파일 라우팅과 안 맞음.
  • Cloudflare Pages "연결"의 범위 — 코드로 준비 가능한 것(wrangler.jsonc, 빌드 설정)까지가 이 브랜치의 몫. 실제 프로젝트 생성(wrangler pages project create bernice 또는 대시보드 Git 연동)은 Cloudflare 계정 인증이 필요한 사용자 단계로 분리(아래 작업 단계 7).
  • P0에 안 넣는 것 — Recharts·TanStack Query 설치(P3에서), 라우팅/레이아웃(P3), 시트 읽기 코드(P1). 홈은 토큰이 보이는 빈 화면 + 헬스체크 표시 수준으로 최소화.

4작업 단계

  1. Vite + React + TS 스캐폴딩을 레포 루트에 생성 — package.json(name bernice), strict TS, 기존 docs/·scripts/·.gitignore 유지·병합 package.json, index.html, vite.config.ts, tsconfig.json, tsconfig.app.json, tsconfig.node.json, src/main.tsx, src/App.tsx, src/vite-env.d.ts
  2. Tailwind v4 설치(tailwindcss, @tailwindcss/vite) + @theme 디자인 토큰(농구 오렌지 스케일·틴트·보조 블루·시맨틱 색) 정의, 빈 홈 화면에 토큰 적용(팀명 헤더 + API 상태 표시) src/index.css, vite.config.ts, src/App.tsx
  3. Pages Functions 뼈대 — GET /api/health 헬스체크 라우트 + Functions 전용 tsconfig + @cloudflare/workers-types functions/api/health.ts, functions/tsconfig.json, package.json
  4. wrangler.jsonc 작성(name · pages_build_output_dir: "./dist" · compatibility_date · vars.SHEET_ID) + wrangler devDep 설치 wrangler.jsonc, package.json
  5. 로컬 시크릿 — 커밋용 템플릿 .dev.vars.example(키 3종: GOOGLE_SERVICE_ACCOUNT_KEY · TEAM_PASSCODE · ADMIN_CODE) 작성, 실제 .dev.vars는 메인 체크아웃 secrets/bernice-502106-02d0840dab2f.json을 한 줄 JSON으로 넣어 생성(코드 2종은 placeholder), git check-ignore .dev.vars로 미추적 검증 .dev.vars.example, .dev.vars(gitignored)
  6. 로컬 개발 파이프라인 — vite.config.ts에 /api → localhost:8788 프록시, npm scripts(dev · dev:api · dev:all · build · typecheck) 정리 후 실제 구동 확인(빈 홈 렌더 + /api/health 200) vite.config.ts, package.json
  7. Cloudflare Pages 연결(사용자 단계) — ! npx wrangler login 후 npx wrangler pages project create bernice 또는 대시보드 Git 연동. 코드 쪽 준비는 4단계에서 완료되므로 이 단계 없이도 로컬 검증·머지는 가능 (코드 변경 없음 — Cloudflare 계정 작업)

5리스크

리스크영향완화책
실제 서비스계정 키를 .dev.vars로 복사 — 실수 커밋 시 키 유출높음이미 .gitignore:9에 등재. 작업 단계 5에서 git check-ignore로 검증하고, 커밋 전 git status에 시크릿 파일이 없는지 재확인. 커밋은 .dev.vars.example만.
Tailwind v4는 v3과 설정 문법이 다름(config 파일 → CSS-first) — 구버전 예제 혼입 시 토큰 미적용중간v4 방식(@import "tailwindcss" + @theme)으로만 작성, tailwind.config.js를 만들지 않음. 홈 화면에서 토큰 색상 렌더를 눈으로 확인.
이중 서버(vite 5173 + wrangler 8788) 프록시 설정 오류 — 로컬에서 /api 404/ECONNREFUSED중간포트를 스크립트에 고정하고 server.proxy를 명시. 테스트 계획에서 5173 경유·8788 직접 호출을 각각 확인.
wrangler pages dev dist는 빌드 산출물이 있어야 기동 — 첫 실행 시 dist/ 없음낮음dev:api 스크립트가 선행 빌드를 포함하거나 README/스크립트 주석에 npm run build 선행을 명시.
Pages 프로젝트 생성은 인터랙티브 인증 필요 — 세션이 대신 못 함낮음작업 단계 7로 분리해 사용자 단계로 안내. 로컬 구동·타입체크·빌드는 인증 없이 전부 검증 가능.

6테스트 계획

  • npm run typecheck(tsc -b) — 앱·functions 모두 타입 에러 0.
  • npm run build — dist/ 생성, 경고/에러 없이 종료.
  • npx wrangler pages dev dist --port 8788 — curl http://localhost:8788/api/health가 200 + {"ok":true,...} JSON, http://localhost:8788/이 빈 홈 렌더(Functions·정적 동시 서빙 확인).
  • npm run dev(vite 5173) — 홈에서 농구 오렌지 토큰 색상 렌더 확인, http://localhost:5173/api/health가 프록시 경유로 동일 응답(개발 워크플로 확인).
  • git check-ignore .dev.vars secrets/ 통과 + git status에 .dev.vars·키 파일 미노출 — 시크릿 미추적 회귀 확인.
  • 기존 자산 회귀 — docs/ 4개 HTML·scripts/ 2개가 변경/이동 없이 그대로인지 git diff --stat으로 확인.

7완료 조건

  • Vite + React + TypeScript 프로젝트가 레포 루트에 생성되고 src/·functions/ 추가, 기존 docs/·scripts/ 유지.
  • Tailwind v4 + 농구 오렌지 디자인 토큰(@theme)이 적용된 빈 홈이 렌더된다.
  • wrangler.jsonc(pages_build_output_dir 포함) 작성 완료 — Pages 프로젝트 생성/Git 연동만 하면 배포 가능한 상태. 실제 연결은 사용자 인증 단계로 안내.
  • .dev.vars(서비스계정 키 / 패스코드 / 관리자코드) 생성 + .dev.vars.example 커밋 + gitignore 동작 검증.
  • wrangler pages dev + vite 로컬 구동 확인 — 빈 홈 + /api/health 200 (프록시 경유 포함).
  • 타입체크·빌드 클린, 시크릿 미추적 상태로 커밋 가능.
