값은 PRD와 100% 동일하다. 변수명만 PRD 표기(text/bg)를 그대로 쓰지 않고 ink/canvas로 바꿨다 — Tailwind v4는 --color-* 변수마다 bg-*/text-*/border-* 유틸리티를 전부 생성하므로, --color-text·--color-bg로 지으면 text-text·bg-bg라는 중복 클래스명이 나온다. 의미는 그대로 두고 이름만 다듬었다.

radius 토큰

Tailwind v4는 @theme의 --radius-* 변수마다 rounded-* 유틸리티를 자동 생성한다. --radius-card: 18px(PRD 16~20px 중간값), --radius-badge: 7px(PRD 6~8px 중간값)을 추가해 rounded-card/rounded-badge로 쓴다. 칩(11px)·버튼(13~15px)은 아직 만들어진 컴포넌트가 없어 이번 스코프에서 제외.

PRD 미명시 매핑 — 설계 판단

PlayerStatusPill의 4개 상태는 PRD §06 "상태 뱃지" 표(미입력/일부/완료 전용)에 직접 대응되지 않는다 — 선수 활동 상태(활동/휴식/비대상/탈퇴)는 다른 도메인이다. 기존 구현이 이미 good/warn/neutral/bad 4색을 의미론적으로 매핑해뒀으므로, 새 팔레트에서도 동일 매핑을 유지한다:

  • 활동 → bg-good-tint text-good-strong
  • 휴식 → bg-warn-tint text-warn-strong
  • 비대상 → bg-neutral-tint text-neutral-strong
  • 탈퇴 → bg-bad/10 text-bad — PRD에 "탈퇴 뱃지" 색이 없어 신규 hex를 발명하는 대신 기존 opacity 패턴을 유지

Gauge는 PRD §05가 "전원 달성 시 primary / 미만이면 primary-soft" 조건부 색을 명시하지만, 현재 Gauge 컴포넌트는 그 조건을 받는 prop이 없다(단색 채움). 조건 분기를 추가하는 건 컴포넌트 동작 변경이라 이번 "토큰 교체" 스코프를 벗어난다고 판단 — 색은 primary 고정으로 교체하고 조건부 분기는 열린 항목으로 남긴다(§5).

Pretendard 웹폰트

npm 패키지 pretendard(레지스트리 확인됨, 최신 1.3.9)의 variable 서브셋 pretendard/dist/web/variable/pretendardvariable.css를 채택 — 웨이트 45~920 전체를 woff2 1개 파일(~2MB)로 커버해 font-bold(700)·font-extrabold(800, 입력 숫자 강조용) 등 임의 굵기를 별도 설정 없이 바로 쓸 수 있다. static per-weight CSS(dist/web/static/*.css)는 웨이트별 파일이 늘어나 이번 스코프엔 과함.
