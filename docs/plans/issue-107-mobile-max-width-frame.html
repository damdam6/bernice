화면 최대 넓이(모바일 프레임) — 레이아웃 전역 중앙정렬 max-width 컨테이너

damdam6/bernice 브랜치 feat/107-mobile-max-width-frame ← main 라벨 enhancement 2026-07-21
✓ 구현 완료 — 10개 단계 전부 완료 · typecheck·vitest(681) 그린 · max-w-frame 실측(데스크톱 1440px → 448px 중앙정렬) 확인

1요약

레이아웃 셸(MainLayout·AdminLayout)에 max-width가 없어 넓은 화면에서 콘텐츠가 뷰포트 전폭으로 퍼지고, 각 화면·공통 컴포넌트가 max-w-md를 제각각 붙여 폭 규칙이 파편화돼 있다. Tailwind v4 @theme에 폭 토큰 --container-frame(28rem = 448px, 현행 max-w-md와 동일값)을 두고, 이를 쓰는 공통 MobileFrame 컨테이너를 도입해 두 레이아웃 · 레이아웃 밖 최상위 화면(LoginGate·NotFound) · BottomNav를 하나의 중앙정렬 프레임 폭으로 수렴한다. 폭 값이 현행과 같아 모바일(≤448px)에서는 시각 변화가 없고, 데스크톱에서만 중앙 고정폭으로 바뀐다.

2현황 분석

  • MainLayout: flex min-h-dvh flex-col bg-canvas 아래 콘텐츠 컬럼 flex flex-1 flex-col pb-16뿐 — max-width 없음(src/components/layout/MainLayout.tsx:6).
  • AdminLayout: 동일한 min-h-dvh flex-col bg-canvas + 헤더(BackButton) 구조, max-width 없음(src/components/layout/AdminLayout.tsx:6).
  • BottomNav: fixed inset-x-0 bottom-0라 뷰포트 전폭 — border-t·bg-white가 전폭으로 늘어남(src/components/layout/BottomNav.tsx:20).
  • 페이지들은 자체 max-width가 없다: Home·Rankings·Players·admin 기록 화면은 px-4/5 py-6만 두고 폭을 부모(레이아웃)에 위임 — 지금 부모가 전폭이라 데스크톱에서 실제로 전폭으로 렌더된다(Home.tsx:44, Rankings.tsx:62, Players.tsx:76, RecordsDateSelect.tsx:75 등).
  • 카드형 화면만 각자 w-full max-w-md: PasscodeGate(LoginGate.tsx:72), NotFound(NotFound.tsx:6), AdminLogin(AdminLogin.tsx:45), SheetManagementHome 내부 컬럼(SheetManagementHome.tsx:29).
  • 공통 카드 컴포넌트도 각자 max-w-md: ErrorPanel(ErrorPanel.tsx:14), EmptyState(EmptyState.tsx:15).
  • 라우팅: 3탭은 LoginGate(내부에서 MainLayout 렌더) 안, 관리자 스택은 AdminLayout, NotFound는 path="*"로 어느 레이아웃에도 속하지 않는다(src/App.tsx). → 폭 규칙을 한 곳(레이아웃)에만 걸면 NotFound·게이트 상태는 누락되므로 공통 컨테이너가 필요.
  • 폭 토큰 부재: src/index.css의 @theme에 색·radius 토큰만 있고 컨테이너 폭 토큰이 없다. Tailwind v4에서 max-w-md는 기본 --container-md(28rem)를 참조한다.

3설계

채택안 — 폭 토큰 + 공통 컨테이너 컴포넌트 조합(이슈의 "컨테이너 컴포넌트 또는 토큰으로 일원화"를 둘 다 충족).

  • ① 폭 토큰: @theme에 --container-frame: 28rem; 추가 → v4의 --container-* 네임스페이스가 max-w-frame 유틸을 생성한다. 폭의 유일 출처가 되며, 값이 현행 max-w-md와 같아 ≤448px에서는 픽셀 무변화.
  • ② 공통 컨테이너: MobileFrame 컴포넌트(mx-auto w-full max-w-frame + className passthrough) — 중앙정렬 컬럼의 단일 구현. 화면별 레이아웃 의도(flex-1 flex-col, pb-16 등)는 className으로 주입.
  • ③ 바깥 배경은 전폭 유지: bg-canvas min-h-dvh는 그대로 뷰포트 전폭 → 데스크톱에서 프레임 밖은 canvas 회색. 별도 레터박스 색을 정할 필요가 없어 §09 "넓은 화면은 중앙 정렬 폭 제한으로만 대응"에 정확히 부합.
  • ④ BottomNav: fixed inset-x-0 bottom-0는 위치용 투명 셸로 두고, 그 안에 MobileFrame로 감싼 실제 바(border-t·bg-white·safe-area pad·탭 flex)를 중앙정렬 → 프레임 폭과 정확히 일치(전폭 아님).

기각안

  • 페이지마다 max-w 추가 → 파편화 지속, 이슈의 dedup 목표에 정면 위배.
  • 컨테이너 쿼리/미디어쿼리 기반 태블릿·데스크톱 반응형 레이아웃 → §09 비스코프(전용 레이아웃 안 함)와 충돌, 과설계.
  • 토큰 없이 max-w-md 문자열만 표준화 → 폭 값이 코드 곳곳에 흩어져 "일원화" 미달. 토큰이 정본 한 곳을 만든다.

4작업 단계

  1. 폭 토큰 추가 — @theme에 --container-frame: 28rem;(주석: 448px, 모바일 프레임 폭 · §01/§09). src/index.css ↳ 검증: dist CSS에서 .max-w-frame{max-width:var(--container-frame)} + --container-frame:28rem 컴파일 확인.
  2. 공통 컨테이너 컴포넌트 생성 — MobileFrame: mx-auto w-full max-w-frame + 선택적 className 병합. 짧은 주석으로 "프레임 폭의 단일 사용처"임을 명시. src/components/layout/MobileFrame.tsx
  3. MainLayout에 프레임 적용 — 콘텐츠 컬럼을 <MobileFrame className="flex flex-1 flex-col pb-16">로 교체. 바깥 bg-canvas min-h-dvh 전폭 유지. src/components/layout/MainLayout.tsx
  4. AdminLayout에 프레임 적용 — 헤더(BackButton)+Outlet을 <MobileFrame className="flex flex-1 flex-col">로 감싸 헤더까지 같은 폭. src/components/layout/AdminLayout.tsx
  5. BottomNav 중앙정렬 — fixed 셸은 투명 위치용으로 두고 내부 MobileFrame 바에 border-t·bg-white·safe-area·탭 flex 이동(전폭 → 중앙 고정폭). src/components/layout/BottomNav.tsx
  6. 레이아웃 밖 최상위 화면 수렴 — LoginGate(PasscodeGate)와 NotFound에서 자체 max-w-md → max-w-frame 토큰으로 교체, 동일 폭 규칙 공유. LoginGate error 분기는 ErrorPanel(8단계에서 max-w-frame화)이 이미 폭을 담당해 추가 변경 불필요. src/components/auth/LoginGate.tsx, src/pages/NotFound.tsx ↳ 계획 조정: 단일 카드 화면은 MobileFrame로 감싸는 대신 카드에 max-w-frame 토큰을 직접 부여. 폭 정본은 토큰 하나로 동일하게 수렴하면서 불필요한 래퍼 노드를 줄임. MobileFrame 컴포넌트는 동적 컬럼(레이아웃 셸·BottomNav) 전용으로 유지.
  7. 관리자 카드 화면 dedup — AdminLogin의 <main>·SheetManagementHome 내부 컬럼의 max-w-md → max-w-frame으로 통일. src/pages/admin/AdminLogin.tsx, src/pages/admin/SheetManagementHome.tsx
  8. 공통 카드 토큰 통일 — ErrorPanel·EmptyState의 max-w-md → max-w-frame(자체 폭 캡은 유지하되 정본 토큰 하나로 수렴). src/components/common/ErrorPanel.tsx, src/components/common/EmptyState.tsx
  9. 테스트 — MobileFrame.test(children 렌더·max-w-frame/mx-auto 클래스·className 병합) 신규, BottomNav.test에 중앙정렬 바(전폭 아님) 검증 추가, MainLayout.test(프레임 안 Outlet + BottomNav 공존) 신규. src/components/layout/MobileFrame.test.tsx, src/components/layout/BottomNav.test.tsx, src/components/layout/MainLayout.test.tsx
  10. 검증 — npm run typecheck · npm run test(83파일·681테스트) 그린. 데스크톱 폭 확인은 실측으로 대체: 실제 컴파일된 .max-w-frame을 1440px 부모 안에서 측정 → 폭 448px·좌우 여백 496px 동일(중앙정렬). 임베디드 브라우저 뷰포트가 448px를 못 넘겨 스크린샷 대신 실측 채택. 전체

병렬 진행 중인 #103(Home)·#104(Players)와의 시각 중복 머지 시 페이지 내부 max-w 잔존 시 이중 제한 두 페이지 모두 자체 max-w가 없음(확인 완료) → git·시각 충돌 없음. 머지 후 육안 재확인.
