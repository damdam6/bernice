1요약

개인 프로필·전체 추이 화면(P4)이 그대로 소비할 PlayerSummary.trends/personalBests를 만드는 순수 함수를 작성한다. 입력은 shared/domain.ts(이슈 #21, main에 PR #16으로 이미 머지됨)의 Player/EventDefinition/Session이고, 이 타입은 그대로 재사용한다 — 이슈 본문의 "#3"은 번호 재부여 전 구번호로 지금의 #21과 동일 이슈다. 시트 파서(#25~#27)가 아직 없으므로 픽스처로 대체해 병렬 개발한다.

3설계

파일 위치 · 함수 시그니처

functions/lib/player-summary.ts에 다음 함수 하나를 공개 API로 둔다(내부 헬퍼는 비공개):

export function computePlayerSummaries(
  events: EventDefinition[],
  players: Player[],
  sessions: Session[],
): PlayerSummary[]

인자 순서는 RecordsResponse의 필드 순서(events → players → sessions)를 따른다 — 이후 #30(엔드포인트)에서 그대로 스프레드해 넘기기 쉽게.

알고리즘

  1. 선수 필터링: player.status !== '탈퇴'인 선수만 순회(타입 좁히기용 type guard로 처리, as 캐스트 없이). 세션의 entries 자체는 필터링하지 않는다 — 탈퇴 선수 데이터는 애초에 파서 단계(#27)에서 걸러진다는 전제(§02)를 따르고, 이 함수는 "어떤 선수를 출력할지"만 책임진다.
  2. 유효 기록 수집: 선수 × 종목 조합마다 전체 세션을 순회해 entry.scores[event.key].status === 'recorded'인 것만 모은다. 그 세션에 선수의 entry 자체가 없으면(아직 미가입) 조용히 건너뛴다 — unmeasured로도 취급하지 않는다. 입력 sessions가 이미 날짜 오름차순이라는 보장에 기대지 않고 sessionDate 기준으로 직접 정렬한다(방어적).
  3. TrendPoint[] 생성: 정렬된 유효 기록을 순서대로 훑으며 직전 유효 기록과 비교 — deltaFromPrevious = value - previous.value(없으면 null), improved는 direction 반영(낮을수록: delta < 0, 높을수록: delta > 0 — 동률(delta === 0)은 개선 아님으로 false). achieved는 경계값 포함 규칙(§04)을 공통 헬퍼로 판정.
  4. PlayerPersonalBest | null 생성: 유효 기록이 없으면 null(호출부에서 필터링돼 스파스 배열이 됨). 있으면 direction 기준 최고값(낮을수록 = min, 높을수록 = max)을 고른다.
  5. 결과 조립: events[] 순서 그대로 trends(항상 이벤트 수만큼), personalBests는 null 아닌 것만 필터.

결정 사항 — PB 동률 처리(계약에 명시 안 됨, 이 이슈에서 확정)

같은 최고값을 두 번 이상 찍은 경우 PlayerPersonalBest.sessionDate를 그 값을 처음 달성한(가장 이른) 회차로 고정한다. "PB"는 기록을 처음 경신한 순간을 가리킨다고 보는 게 자연스럽고("타이 기록"은 PB를 다시 세우는 게 아니라 기존 PB를 유지하는 것), 나중에 프론트에서 "이 회차에 PB를 세웠다" 배지를 붙일 때도 최초 달성 시점이 더 유용하다. 계약 문서(records-schema.html)에 반례가 없으면 이 규칙을 그대로 채택한다.

4작업 단계

  1. 공통 판정 헬퍼(isAchieved(direction, value, targetValue), isImproved(direction, delta))와 유효 기록 수집(collectValidScores) 작성 — 전부 비공개, computePlayerSummaries 내부에서만 사용 functions/lib/player-summary.ts
  2. buildTrend(선수×종목 → PlayerEventTrend)와 buildPersonalBest(선수×종목 → PlayerPersonalBest | null) 작성 — PB 동률은 최초 달성일 기준 functions/lib/player-summary.ts
  3. computePlayerSummaries 조립 — 탈퇴 제외 type guard, events[] 순서 유지, personalBests 스파스 필터 functions/lib/player-summary.ts
  4. 픽스처 테스트: 방향 양쪽(낮을수록/높을수록) · 경계값 달성 · 면제/미측정/이상값이 낀 시계열의 delta/improved 스킵 · 아직 미가입 선수의 과거 회차 부재 · PB 갱신되는 케이스와 갱신 안 되는(더 나빠진 나중 기록) 케이스 · PB 동률 시 최초 달성일 · 탈퇴 선수 완전 제외 · 비대상/휴식 선수는 정상 포함 · 기록 전혀 없는 종목은 points: []이고 personalBests에 항목 자체가 없음 functions/lib/player-summary.test.ts
