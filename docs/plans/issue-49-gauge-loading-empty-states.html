1요약

이슈 #49의 원 구현(PR #53)은 이미 origin/main에 스쿼시 병합돼 있고, 이 브랜치의 기존 커밋 2개는 그 내용과 컴포넌트 파일 기준 100% 동일합니다. 이번 계획 세션에서 브랜치를 origin/main(#75 인디고 테마 개편 포함, 14커밋 최신)으로 리셋해 동기화를 마쳤습니다. 남은 작업은 2026-07-17 디자인 인터뷰로 확정된 게이지 시각 스펙 — 원형 SVG 링 대신 8px 트랙 + 채움 바, "n/m명 달성" 라벨, 전원 달성 색 분기, width .5s 트랜지션 — 을 반영해 Gauge 컴포넌트를 재설계하는 것입니다. 3설계

  • props를 카운트 쌍으로 교체 — value: number(0~100 퍼센트) 단일값 대신 achievedCount: number, eligibleCount: number. 이유: (a) 새 라벨 "n/m명 달성"은 퍼센트 하나로는 복원 불가능한 두 정수가 필요, (b) §05가 명시하는 실제 소비 데이터 모양과 1:1 대응시켜 P4에서 변환 코드 없이 그대로 꽂을 수 있음, (c) "전원 달성" 색 분기(eligibleCount>0 && achieved>=eligible)도 카운트 쌍에서 자연스럽게 유도됨. 도메인 타입(EventAchievementRate)은 여전히 import하지 않아 결합 없음을 유지.
  • size(sm/md) variant 제거 — 스펙이 8px 트랙 고정값 하나뿐이고 확인된 사용처가 홈 화면 1곳뿐이라 불필요한 분기를 없앰(원 PR의 sm/md는 원형 링 시절 가상의 재사용성 대비였음). 필요해지면 나중에 다시 추가.
  • 렌더링 교체 — 원형 SVG 링(viewBox, strokeDasharray)을 제거하고 8px 높이(h-2) div 2겹(bg-gauge-track 트랙 + bg-primary/bg-primary-soft 채움 바, width: {percent}%)로 대체. 순수 계산은 gauge-math.ts에 남겨(computeGaugePercent·isGaugeFullyAchieved·formatGaugeLabel 3개) 원 PR의 "DOM 없이 단위 테스트" 원칙을 계승.
  • 접근성 — role="img" → role="progressbar" + aria-valuenow/aria-valuemin/aria-valuemax로 교체(진행률 표시에 더 맞는 시맨틱).
  • 애니메이션 — 스펙(docs/prd-design.html §03 모션 표: "게이지 width .5s")대로 width만 transition-[width] duration-500로 트랜지션. 색 변경 자체는 트랜지션 없음.
  • 방어적 처리 — eligibleCount ≤ 0(전원 면제 등)이면 0%/미달성 색으로 렌더(도메인 규칙 "eligibleCount 0 → rate 0"과 동일 결). achievedCount는 계산용으로는 [0, eligibleCount] 클램프, 라벨 표시는 NaN만 0으로 가드해 원값을 그대로 보여줌(원 PR의 clampGaugeValue NaN 가드 철학 계승).
  • Spinner/ErrorPanel/EmptyState — 변경 없음. 코드 diff 0으로 이미 인디고 토큰 기반임을 확인했고, 브라우저 렌더 재확인만 수행.

4작업 단계

  1. gauge-math.ts를 카운트 기반 순수 함수로 재작성 — computeGaugePercent(achievedCount, eligibleCount)·isGaugeFullyAchieved(achievedCount, eligibleCount)·formatGaugeLabel(achievedCount, eligibleCount) 3개로 교체. 기존 원형 상수(GAUGE_RADIUS·GAUGE_CIRCUMFERENCE)와 getGaugeDashArray는 제거. src/components/common/gauge-math.ts
  2. gauge-math.test.ts를 새 함수 기준으로 재작성 — 0명/중간/전원 달성, eligibleCount=0(전원 면제), achievedCount가 eligibleCount 초과(방어적 클램프), NaN 케이스를 커버. src/components/common/gauge-math.test.ts
  3. Gauge.tsx를 원형 SVG 링에서 선형 바로 재작성 — props를 value 단일값에서 achievedCount/eligibleCount/label?로 교체. 8px bg-gauge-track 트랙 위에 bg-primary(전원 달성)/bg-primary-soft(미만) 채움 바를 width + transition-[width] duration-500로 렌더하고, "n/m명 달성" 라벨과 role="progressbar" 접근성 속성을 붙인다. src/components/common/Gauge.tsx
  4. Gauge.test.tsx 신설 — @testing-library/react로 라벨 텍스트("n/m명 달성"), 전원 달성 시 bg-primary 클래스, 미달 시 bg-primary-soft 클래스, width 스타일 값(0%/50%/100%), eligibleCount=0 엣지 케이스를 렌더 검증(RankingTable.test.tsx 패턴 준용). src/components/common/Gauge.test.tsx ✓ 완료 — typecheck·vitest 267개 전부 통과 확인

5리스크

리스크영향완화책
Gauge가 아직 어디에도 연결되지 않아(P4 홈 화면은 별도 이슈) 실제 사용 시 API가 재조정될 수 있음낮음 — 재작업이 있어도 소규모§05 스펙의 achievedCount/eligibleCount 필드명과 정확히 맞춰 설계해 P4에서 변환 없이 그대로 소비 가능하게 함
브랜치를 origin/main으로 리셋해 원격(origin/feat/49-gauge-loading-empty-states)과 히스토리가 갈라짐(14개 vs 2개 커밋)이후 git push가 non-fast-forward로 거부됨구 커밋 2개가 이미 main에 동일 내용으로 존재함을 diff로 확인함 — PR 오픈 시 이 브랜치로 force-push 필요(개인 저장소의 미병합 feature 브랜치라 안전, main/공유 브랜치 아님)
확정 목업 스크린샷이 로컬에 없음(claude.ai/design 프로젝트 소유, docs/prd-design.html §01)낮음 — 여백·정렬 등 텍스트 스펙 밖 세부는 판단 필요텍스트 스펙(트랙 색·높이·채움 색·라벨·트랜지션)은 정확히 구현하고, 세부 여백은 기존 카드/뱃지 radius·spacing 관례를 따르며 필요시 리뷰에서 조정
