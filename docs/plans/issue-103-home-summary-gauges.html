1요약

src/pages/Home.tsx는 아직 P0 스캐폴딩 스텁이다 — /api/health만 핑하고 정적 placeholder 카드를 그린다. 홈이 그릴 데이터(RecordsResponse.home)와 게이지 컴포넌트(AchievementGauge)는 이미 구현·테스트가 끝났으나 사용처가 0이다. 이번 작업은 스텁을 걷어내고, LoginGate가 이미 캐시에 채워둔 useRecords() 데이터로 헤더(BERNICE + 잠금) · 인디고 최신회차 카드 · 종목별 달성률 게이지 목록 · 바로가기 2카드를 docs/prd-design.html §05~§07와 DesignSync 목업 HOME 섹션 그대로 조립한다. 파생값(n차 라벨·평균 달성%)과 게이지 매핑은 순수 함수로 분리해 단위 테스트하고, 화면 렌더는 컴포넌트 테스트로 못박는다.

2현황 분석

  • 현재 홈은 P0 스텁. src/pages/Home.tsx:20-31이 /api/health를 핑해 API 상태 pill만 그리고, src/pages/Home.tsx:46에 "P0 스캐폴딩 · 화면은 P3~P4에서 채워집니다" 문구가 박혀 있다. 실제 회차/달성률은 전혀 없다.
  • 데이터는 이미 준비돼 있다. src/components/auth/LoginGate.tsx:17가 useRecords()로 /api/records를 페치하고, 성공(비-401·비-에러) 시에만 MainLayout(→ Outlet → Home)을 렌더한다 (LoginGate.tsx:19-41). 따라서 홈이 useRecords()를 다시 부르면 staleTime: 5분(src/hooks/useRecords.ts:61) 캐시 히트로 즉시 데이터를 얻는다 — 재요청 없음.
  • 홈 요약 계약. HomeSummary(shared/domain.ts:133-139)는 latestSession: { date, participantCount } | null과 achievementRates: EventAchievementRate[](event/achievedCount/eligibleCount/rate, domain.ts:126-131)를 준다. events[]·sessions[](날짜 오름차순)도 같은 응답에 있다(domain.ts:142-149).
  • 게이지 컴포넌트는 완성품. src/components/charts/achievement-gauge.tsx:10-26는 value(0~1)만 받아 §07 스펙(8px bg-gauge-track 트랙, 전원 달성 bg-primary / 미만 bg-primary-soft, transition-[width] duration-500)대로 그린다. 종목명·"n/m명 달성" 텍스트는 화면 책임(주석에 명시). 배럴 src/components/charts/index.ts:1로 { AchievementGauge } export.
  • 참고할 형제 화면. src/pages/Rankings.tsx:13-43가 로딩/에러/빈 상태 3분기 패턴 (isError→ErrorPanel, !data→Spinner, 데이터 0건→EmptyState)과 n차 라벨 규칙(Rankings.tsx:76 — sessions.map((s,i)=> …{i+1}차))을 이미 쓴다. 홈도 같은 규칙을 따른다.
  • 토큰·라우팅 확인. 색은 전부 src/index.css @theme 토큰 (--color-primary #3f51b5:12, --color-primary-soft #c5cae9:15, --color-gauge-track #eef0f4:47, --color-ink-sub:26, --color-neutral-strong #9096a2:39)으로 존재한다. 라우트는 /admin/login·/rankings·/players가 src/App.tsx에 모두 있다(잠금·바로가기 목적지).

3설계

정본은 docs/prd-design.html §05 🏠 홈 표(요소→필드) + §06(공통 컴포넌트) + §07(게이지), 시각 배치는 DesignSync 목업 버니스 기록.dc.html HOME 섹션이다. 아래 매핑·색·수치는 목업에서 직접 읽어 확정했다.

파생 로직 → 순수 함수로 분리 (src/lib/home-summary.ts)

  • n차 라벨: sessions(날짜 오름차순)에서 latestSession.date의 index+1 → "3차". Rankings의 {i+1}차와 동일 규칙. latestSessionOrdinal(sessions, date).
  • 평균 목표 달성%: achievementRates[].rate 단순 평균 ×100 반올림. 목업 계산(Math.round(mean(count/total)*100))과 동치 — rate가 곧 achievedCount/eligibleCount(0 가드 포함). 빈 배열이면 0. averageAchievementPct(rates).
  • 게이지 목록: achievementRates[]를 순회하며 라벨은 events[]에서 event key로 조회(누락 시 key fallback). { label, achievedCount, eligibleCount, rate }[]. buildHomeGauges(rates, events).

화면 구조 (src/pages/Home.tsx)

  • 3분기 게이트: Rankings와 동일하게 useRecords()로 isError→CenteredPanel+ErrorPanel, !data→CenteredPanel+Spinner. (LoginGate가 이미 걸러 사실상 방어 코드지만 형제 화면과 일관.)
  • 헤더(항상): 좌 BERNICE 워드마크(30px/800, uppercase, text-primary, letter-spacing .03em) + 우 잠금 버튼(40×40, radius 12, border-input-line, hover border-primary) → <Link to="/admin/login"> + lucide Lock, aria-label="관리자 로그인".
  • 빈 상태: latestSession === null이면 헤더 아래 EmptyState title="아직 기록된 회차가 없습니다" (Rankings 카피 재사용). 헤더는 남겨 관리자 로그인 진입을 유지.
  • 인디고 최신회차 카드(bg-primary rounded-[20px] px-[22px] py-5 text-white): 상단 좌 "최신 회차 · {n차}"(text-primary-soft 13px/600) / 우 {date}(20px/800, 흰색, YYYY-MM-DD 원본); 하단 우측 2스탯 — {participantCount}명 "참여 인원" / {avg%}% "평균 목표 달성" (값 22px/800 흰색, 단위 text-primary-soft, 라벨 11px). 카드 radius는 목업의 20px 준수(공용 Card의 18px·p-6·shadow와 불일치해 인라인).
  • 게이지 목록: "종목별 팀 목표 달성률" 헤딩(14px/700 text-ink) 아래 각 행(흰 배경, border-line, radius 16, padding 14/16) = {종목 라벨}(14px/600 text-ink) + "{achievedCount}/{eligibleCount}명 달성"(12px text-ink-sub) + <AchievementGauge value={rate} />. achievementRates.length > 0일 때만 섹션 렌더.
  • 바로가기 2카드: "바로가기" 헤딩 아래 2열 그리드 — <Link to="/rankings"> lucide Trophy + "랭킹" / "누가 제일 잘해?", <Link to="/players"> lucide TrendingUp + "개인 추이" / "종목별 성장 그래프" (아이콘 text-primary, 제목 14px/700, 부제 11px text-neutral-strong, hover border-primary).

색 토큰 판단

  • 목업 #C5CAE9(단위·"최신 회차·n차") = 기존 primary-soft 토큰 그대로 사용.
  • 인디고 카드 위 소형 스탯 라벨 색 #9FA8DA는 토큰 미존재 → 목업 충실도를 위해 인라인 임의값 text-[#9fa8da] 1회 사용(대체안: primary-soft로 통일 시 명도 위계 상실). 구현 시 최종 확정.
