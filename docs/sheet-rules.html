📝 시트 편집 규칙

이 시트를 직접 수정하기 전에 읽어주세요. 몇 가지 규칙만 지키면 사이트가 안 깨집니다.

30초 요약: ① 버니스명단·목표 행 순서 바꾸지 않기(정렬·재배치 금지, 추가만) · ② 탈퇴는 행 삭제 말고 상태만 '탈퇴' · ③ 헤더(1행)·컬럼 순서 그대로 · ④ 새 회차 탭 이름은 YYYY-MM-DD · ⑤ 서비스 계정 공유 유지 · ⑥ 이름·종목명 칸은 참조 수식이라 손대지 않기(점수만 입력)

01절대 규칙 (깨지면 사이트가 망가짐)

🔒 버니스명단 · 목표 행 순서 고정 (정렬·재배치 금지)
버니스명단과 목표는 행을 정렬하거나 위아래로 옮기지 않는다. 새 팀원은 맨 아래로 추가만.
왜: 회차의 이름·종목명은 행 위치로 참조(='버니스명단'!A2)한다. 행을 옮기면 이름은 밀려 바뀌는데 점수는 그대로라 서로 어긋난다. 이름순 정렬 같은 건 사이트에서 하면 된다.
🔒 탈퇴 = 행 삭제 금지, 상태만 '탈퇴'로
버니스명단에서 사람을 지우지 말 것. 나갔으면 상태 드롭다운을 탈퇴로 바꾸기만.
왜: 행을 지우면 그 행을 참조하던 회차 이름이 아래 사람으로 밀려 점수가 어긋난다. 상태만 바꾸면 시트 데이터(과거 기록 포함)는 그대로 보존되고, 사이트에서는 전체 숨김 처리된다 — 과거 회차 기록까지 포함해 사이트 어디에도 나타나지 않는다(상세: 도메인 타입·API 계약 §02).
🔒 이름 · 종목명 칸은 참조 수식 — 손대지 않기
회차 탭의 이름 열과 종목 헤더는 ='버니스명단'!A2·='목표'!A2 같은 수식이다. 직접 타이핑해서 덮어쓰지 않는다. 사람이 채우는 건 점수 숫자뿐.
왜: 참조라서 명단·목표에서 이름/종목명을 바꾸면 회차에도 자동 반영된다. 덮어쓰면 이 연결이 끊긴다.
🔒 헤더(1행)와 컬럼 순서 고정
각 탭의 첫 행 제목과 열 순서를 바꾸지 않는다. 열 추가/삭제/이동 금지.
왜: 사이트가 열 제목·순서를 보고 데이터를 읽는다. 바꾸면 값이 뒤섞이거나 안 읽힌다. (열을 추가하려면 사이트 코드도 같이 고쳐야 함 → 먼저 문의)
🔒 새 회차 탭 이름은 날짜 YYYY-MM-DD
회차 탭 이름은 반드시 2025-08-16 같은 날짜 형식. 버니스명단·목표는 그대로 둔다.
왜: 사이트가 '날짜 형식 탭 = 회차'로 자동 인식하고 시간순 정렬한다. 다른 이름이면 회차로 안 잡힌다.
🔒 서비스 계정 공유 유지
시트 공유에서 bernice-sheet@…iam.gserviceaccount.com의 편집자 권한을 제거하지 않는다.
왜: 사이트는 이 계정으로 시트를 읽는다. 공유가 끊기면 사이트가 데이터를 못 불러온다.
⚠️ 표 중간에 빈 행·메모 행 넣지 않기
데이터 표 안에 빈 줄이나 "여기부터 미참여" 같은 메모 행을 넣지 않는다.
왜: 사이트가 빈 행을 데이터의 끝으로 오해할 수 있다. 메모는 다른 탭이나 셀 옆쪽에.

02자주 하는 작업 ① — 새 회차 추가 (3개월마다)

  1. 가장 최근 회차 탭을 복제 (탭 우클릭 → 복사) — 이름·종목 참조 수식과 서식이 그대로 따라옴
  2. 복제된 탭 이름을 측정한 날짜로 변경 (예: 2025-08-16)
  3. 이전 점수를 지우고 이번 기록의 점수 숫자만 입력. 이름·헤더 칸은 건드리지 않는다(참조라 자동)
  4. 그 회차 미참여자는 점수만 비운다 (행 삭제 X — 이름은 그대로 남겨둠)
  5. 버니스명단에 새 팀원이 늘었으면, 이름 열의 참조 수식을 맨 아래로 한 칸 더 끌어내려 포함(드래그)

✓ 이렇게

  • 기존 회차 복제해서 참조·서식 유지
  • 날짜 형식 탭 이름
  • 점수 숫자만 입력, 면제는 면제

✗ 이러지 말기

  • 이름·헤더 칸에 직접 타이핑(참조 수식 덮어쓰기)
  • 탭 이름을 3차, 8월 등으로
  • 미참여자에게 0점 입력 (0점 ≠ 미참여)

03자주 하는 작업 ② — 팀원 가입 / 탈퇴

새로 가입

  1. 버니스명단 맨 아래에 행 추가 (중간 삽입 X)
  2. 이름 입력, 상태는 드롭다운에서 활동
  3. 다음 회차부터 회차 탭 이름 열 참조를 이 행까지 늘리면 자동 포함

탈퇴

  1. 버니스명단에서 그 사람의 상태를 탈퇴로 변경 (행 삭제·이동 X)
  2. 끝. 시트 데이터(과거 기록 포함)는 보존되지만, 사이트에서는 과거 회차 기록까지 포함해 전체 숨김 처리된다

※ 나갔다가 다시 들어오면 → 상태만 활동으로 되돌리기 (같은 행 유지, 추이가 이어짐).
