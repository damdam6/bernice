01프로젝트 개요 & 운영 프로세스

버니스(농구팀)의 3개월 주기 실력 향상 테스트 기록을 관리·시각화하는 사이트입니다. 데이터는 구글 스프레드시트 한 개에 회차별 탭으로 누적하고, 사이트는 그 시트를 참조합니다.

운영 흐름

구글 스프레드시트 (1개) ├─ 탭: 2025-05-16 ← 1차 (현재 데이터) ├─ 탭: 2025-08-16 ← 2차 (3개월 뒤, 동일 형식으로 탭만 추가) └─ 탭: ... ← 회차마다 탭 추가 │ ▼ 사이트가 시트를 자동으로 읽어 반영 기록 사이트

핵심 원칙: 사용자는 탭 추가만 하면 되고, 탭 이름을 코드에 하드코딩하지 않고 자동 탐색하여 새 회차가 자동 반영됩니다.

04사이트 구조도

🏀 버니스 실력 기록 모바일 우선 · 하단 네비 3탭 (2026-07-17 디자인 개편) │ ├─ 🏠 홈 "이번 분기 한눈에" │ ├─ 최신 회차 카드 (n차·날짜 / 참여 인원 / 평균 목표 달성 %) │ ├─ 종목별 팀 목표 달성률 게이지 (달성 n/m명) │ ├─ 바로가기 (랭킹 / 개인 추이) │ └─ 🔒 관리자 로그인 진입 (우상단 잠금 아이콘) │ ├─ 🏆 랭킹 "누가 제일 잘해?" │ ├─ 종목 칩 (셔틀런/골밑슛/자유투/패스캐치) + 회차 칩 (1차/2차/…) │ └─ 순위 행 — 목표 달성 뱃지 · 미측정/면제 선수도 하단 노출 │ ├─ 👤 개인 "내 성장 추이" · 전체 추이 페이지 통합 │ ├─ 선수 드롭다운 + 회차 칩 │ ├─ 레이더 차트 (4종목 스킬 프로필) │ └─ 종목별 성장 카드 (현재값 · PB · 전 회차 대비 ▲▼) │ └─ 탭 → 추이 차트 확장: 전체 선수 배경 라인 + 본인 하이라이트 + 목표선 │ └─ 🔒 관리자 · 시트 관리 "운영자 전용 · 관리자 코드" ├─ 기록 입력 — 날짜 선택(완료 n/N) → 참가자 목록(미입력/일부/완료) → 선수별 입력 ├─ 참가자 추가 — 기존 회차에 활동 선수 추가(맨 아래) └─ 기록지 만들기 — 오늘(KST) 회차 탭 생성 (참가자 선택: 활동만·가나다·기본 해제)

🔒 관리자의 세 기능은 시트에 쓰기를 하며, 관리자 세션으로 보호. 자세한 스펙은 아래 05 참고. 열람(홈·랭킹·개인)은 팀 패스코드 게이트 유지 — 게이트 화면은 디자인에 추가 예정(디자인 PRD §09). 기존 "📈 전체 추이" 별도 페이지는 2026-07-17 디자인 개편에서 삭제 확정 — 개인 탭의 추이 차트로 통합.

05관리자 페이지 "시트 관리" — 기능 스펙

사이트 → 시트 쓰기 기능. 관리자 세션으로 보호, 성공 시 캐시 자동 무효화. 시트 관리 홈의 버튼 순서는 기록 입력(강조) → 참가자 추가 → 기록지 만들기 (2026-07-17 디자인 확정).

① 기록지 만들기

항목동작
날짜오늘 (Asia/Seoul), YYYY-MM-DD. 오늘 탭 있으면 생성 안 함 + 안내
참가자 선택활동 선수만, 가나다 오름차순, 기본 전원 해제 → 체크한 사람만 탭에 생성
생성 방식최근 탭 복제 → 이름열을 선택 참가자 참조(가나다)로 재작성 → 점수 비움
원자성한 번의 batchUpdate — 실패 시 전체 롤백(반쪽 탭 없음)
이후"시트 열기" 링크 표시. 점수는 사이트 기록 입력(아래 ③)으로 — 시트 직접 입력은 escape hatch로 계속 허용

② 기존 기록지에 사람 추가하기

항목동작
대상 회차날짜 탭 선택 (기본 = 최근)
대상 사람그 탭에 없는 활동 선수 멀티선택
위치맨 아래에 추가 (이름 참조 + 빈 점수)

③ 기록 입력 — 점수 입력·수정 (PRD 확정 · 상세 문서)

항목동작
권한관리자 세션 전용 (팀 패스코드는 열람만)
동선날짜 선택(회차 카드·완료 n/N) → 참가자 목록(가나다·미입력/일부/완료 뱃지) → 선수별 입력 — 시간 = 분:초 2필드 / 개수 = 스테퍼(−/+)+숫자 키패드 / 면제 토글(면제 가능 종목만) / 빈칸 = 미측정
저장선수 단위 — 회차 탭 행 1개 업데이트. 자유 수정(과거 회차 포함), 동시 입력은 마지막 저장 승리. 저장 후 목록 복귀 + 토스트
검증normalize-score 규칙 재사용 — 형식 오류(invalid)는 시트에 쓰지 않고 400 거부
이후저장 성공 시 records 캐시 무효화 → 팀원 열람에 즉시 반영
