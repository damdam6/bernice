P1 백엔드 데이터 레이어 (읽기) 가장 핵심

의존: P0
  1. 공용 서비스계정 JWT 서명 유틸 (Workers Web Crypto)
  2. Sheets API 읽기 래퍼 (액세스 토큰 캐시)
  3. 탭 분류: 명단(버니스명단) / 목표 / 회차(날짜) — NFC 정규화 주의
  4. 파서: 명단(이름·상태) · 목표(종목·목표·만점·방향) · 회차(이름·점수)
  5. 값 정규화: 1:15→초 · 면제 · 빈칸 · 상태 4종
  6. 종목별 랭킹 계산 (방향 반영: 셔틀런 낮을수록 / 면제·빈칸 제외)
  7. 개인 추이 · PB · 목표 달성 계산
  8. /api/records 엔드포인트 + JSON 응답 계약 확정 + 긴 캐시(Cache API)

P2 인증 팀 전용 게이트

의존: P1
  1. /api/login — 패스코드 검증 → HttpOnly 쿠키
  2. 인증 미들웨어 — records/refresh 보호 (쿠키 없으면 401)
  3. /api/refresh — 캐시 무효화 (로그인 필요)
  4. 로그인 게이트 UI (프론트)

P3 프론트 공통 뼈대 토대

의존: P1 (JSON 계약), P2
  1. 라우팅 + 레이아웃 + 네비게이션
  2. TanStack Query 셋업 + /api/records fetch 훅
  3. 공통 컴포넌트 (카드 / 뱃지 / 게이지 / 테이블 / 차트 래퍼)

P4 화면 (읽기 기능) 사용자 화면

의존: P3
  1. 🏠 홈 — 최신 회차 요약(참여인원/날짜) + 종목별 목표 달성률 게이지
  2. 🏆 랭킹 — 종목별 랭킹 + 회차 선택 + 목표 달성 뱃지
  3. 👤 개인 프로필 — 선수 선택 / 레이더 / 종목별 추이 라인 / PB / 전회차 대비 변화
  4. 📈 전체 추이 — 전체 인원 × 회차 멀티라인·매트릭스 + 종목 전환

P5 관리자 페이지 (쓰기) 운영자 전용

의존: P1(파서·탭유틸 재사용), P2
  1. 관리자 코드 검증 유틸
  2. /api/admin/create-sheet — 원자적 batchUpdate(복제 → 참가자 참조 재작성 가나다 → 점수 비움), 오늘(KST)·중복 가드
  3. /api/admin/add-players — 대상 회차에 활동 선수 맨 아래 추가
  4. 🔒 관리자 페이지 UI — 기록지 만들기(참가자 선택 모달: 활동만·가나다·기본 해제) + 사람 추가 폼
  5. 성공 시 캐시 무효화 + "시트 열기" 링크
